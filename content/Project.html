專案簡介：分散式 Reveal.js 簡報編輯系統

這是一個使用 Flask 所建立的 Web 編輯器，用於多人協作編輯 Reveal.js 簡報內容（以 Markdown 撰寫），並透過 Git 作為儲存與版本控制工具。每位使用者可在本地端啟動編輯介面，修改後使用自己的 GitHub Token 或 SSH 金鑰來 git push 到遠端分支，藉此保留作者身分與提交記錄。

整個系統的設計理念為：

  1. 本地運行：使用者 clone 後可在本地執行 Flask 網頁伺服器。
  2. Web 編輯介面：提供簡單的 Markdown 編輯器與 Reveal.js 預覽。
  3. Git 管理版本：使用者使用自己的 Git 身份（token 或 SSH）將內容推送到遠端分支。
  4. 非同步協作：統一合併簡報內容透過 Pull Request 或 Git 合併流程。
  5. 這套系統不需要後端帳號認證與 Git 操作，而是透過 GitHub 分支和 PR 工作流，達成簡潔、可追蹤、模組化的多人協作。

啟動本地 Flask Web 編輯器

功能描述：使用者在自己的電腦上執行 Flask 應用，開啟 localhost 編輯介面。

app.py

from flask import Flask, render_template, request, redirect, url_for
import os

app = Flask(__name__)
SLIDES_PATH = os.path.join("slides", "slides.md")

@app.route("/", methods=["GET", "POST"])
def editor():
    if request.method == "POST":
        markdown = request.form["markdown"]
        with open(SLIDES_PATH, "w", encoding="utf-8") as f:
            f.write(markdown)
        return redirect(url_for("editor"))

    with open(SLIDES_PATH, "r", encoding="utf-8") as f:
        markdown = f.read()
    return render_template("editor.html", content=markdown)

if __name__ == "__main__":
    app.run(debug=True, port=5000)

前端 Reveal.js + Markdown 編輯器

功能描述：提供網頁介面讓使用者編輯 Markdown，並即時預覽 Reveal.js 簡報。

templates/editor.html

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>簡報編輯器</title>
  <script src="https://cdn.jsdelivr.net/npm/reveal.js@4/dist/reveal.min.js"></script>
  <script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/reveal.js@4/dist/reveal.css">
  <style>
    textarea { width: 100%; height: 300px; margin-bottom: 1em; font-family: monospace; }
  </style>
</head>
<body>
  <h1>Markdown 編輯器</h1>
  <form method="POST">
    <textarea name="markdown">{{ content }}</textarea>
    <br>
    <button type="submit">儲存</button>
  </form>

  <hr>
  <h2>Reveal.js 預覽</h2>
  <div class="reveal">
    <div class="slides" id="slides"></div>
  </div>

  <script>
    const markdown = document.querySelector("textarea").value;
    const slidesDiv = document.getElementById("slides");

    const slides = markdown.split(/^---$/m);
    const html = slides.map(md => `<section>${marked.parse(md)}</section>`).join('');
    slidesDiv.innerHTML = html;

    Reveal.initialize();
  </script>
</body>
</html>

Git 操作由使用者本地負責（非後端）

功能描述：使用者在本地透過 CLI 執行 Git 操作，使用自己的身份進行 commit 與 push。

指令建議（你可印在前端或 README 中）

# 取得最新版本
git pull origin main

# 建立新的分支
git checkout -b users/your-name-update

# 加入修改的檔案
git add slides/slides.md

# 提交變更
git commit -m "新增第 5 頁投影片"

# 推送到遠端分支（使用者自己的 Git 身份）
git push origin users/your-name-update

# 接著到 GitHub 建立 Pull Request

使用者操作指令

git checkout -b users/你的名字-20250926
git add slides/slides.md
git commit -m "更新簡報內容"
git push origin users/你的名字-20250926
